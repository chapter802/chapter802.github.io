source map查错

  • devtool:‘inline-source-map’ //位置 webpack.config.js

代码自动化编译

  1. 使用观察模式
    • 方法: package.json 文件script部分添加"watch": "webpack --watch"
  2. 使用 webpack-dev-server
    • 方法: 修改webpack.config.js告诉开发服务器(dev server)，在哪里查找文件：devServer:{contentBase: './dist'}, 在pakage.json文件script部分添加"start": "webpack-dev-server --open",然后直接运行npm start
  3. 使用 webpack-dev-middleware

模块热替换 只适合生产环境

生产环境构建

  1. 自动方式

    • webpack-p相当于webpack --optimize-minimize --define process.env.NODE_ENV="'production'"

      • 用UglifyJsPlugin压缩js,
      • 运行LoaderOptionsPlugin,
      • 设置 NodeJS 环境变量，触发某些 package 包，以不同的方式进行编译。

  2. 手动方式

    • 定义两个完全独立的配置文件
      webppack.dev.js

      webpack.prod.js

      pakage.json

      webpack.config.js

      了解 env

      • 高级途径 详见>>
        一个基本配置文件，包含两个环境通用的配置，然后将其与特定于环境的配置用 webpack-merge 进行合并。这将为每个环境产生完整配置，并防止重复公共部分代码。

Tree Shaking

  • 从maths.js 引入cube方法 import {cube} from './maths.js';

代码分离

  1. 入口起点：使用 entry 选项手动分离代码。
    缺点: 会引起代码重复, 动态分离代码缺乏弹性.
  2. 防止重复：使用 CommonsChunkPlugin 去重和分离 chunk。
    • ExtractTextPlugin : 分离CSS
    • bundle-loader : 分离代码,懒加载结果包
    • promise-loader : 类似bundle-loader,用promises方法

  3. 动态导入：通过模块的内联函数调用来分离代码。

    • AsyncFunciton new AsyncFunction([arg1[, arg2[, ...argN]],] functionBody)

  4. Bundle Analysis
    webpack-chart ,webpack-visualizer .

缓存

确保编译好的文件能被客户端缓存, 文件内容变化后能够请求到新文件.

  • 输出文件名 //通过 output.filename 使用 [chunkhash] 替换 filename: '[name].[chunkhash].js' . 将第三方库(library)（例如 lodash 或 react）提取到单独的 vendor chunk 文件中，是比较推荐的做法.
  • 提取模板 // CommonsChunkPlugin
  • 模块标识符 // HashedModuleIdsPlugin 用于生产环境
